Architecture / Von Neumann / CPU
2016 Q3 2016 8 marks ▼
Describe the function of the main components of the Von Neumann CPU architecture. [8]
▶ Mark Scheme
Award 1 mark for naming + 1 mark for describing each component (up to 8):
  • Control Unit / Clock — Manages the Fetch/Decode/Execute cycle; fetches each instruction in sequence, decodes and synchronises it by sending control signals to other parts of the computer.
  • Arithmetic Logic Unit (ALU) — The processing and manipulation of data which normally consists of arithmetic operations or logical comparisons, allowing a program to take decisions.
  • Registers (MAR/MDR/CIR etc) — Small/fast-access/temporary storage typically addressed by mechanisms other than main memory; or a description of a specific register.
  • Buses (Data Bus / Control Bus / Address Bus) — Connects all internal components of a computer, such as CPU and memory, to the motherboard; or specific role of named bus.
  • Cache Memory (L1, L2) — Can be accessed by a CPU more quickly than regular RAM to execute frequently accessed items of code (e.g. loops).
2019 Q3 2019 6 marks ▼
Apart from buses, describe the function of three main components in a contemporary Von Neumann type CPU architecture. [6]
▶ Mark Scheme
Award 1 mark for naming + 1 mark for describing each of three components:
  • Control Unit — Manages the fetch/decode/execute cycle; sends control signals to other parts of the computer.
  • ALU — Carries out arithmetic operations and logical comparisons.
  • Registers — Small, fast temporary storage. MAR holds memory address; MDR holds data being transferred; CIR holds current instruction being decoded.
  • Cache — Holds frequently accessed instructions/data; faster to access than main RAM.
2022 Q1 2022 4 marks ▼
Describe the fetch-decode-execute cycle showing how data can be read from RAM into registers. Your answer should make reference to the MAR, MDR, PC and CIR. [4]
▶ Mark Scheme
One mark for each of the following:
  • Fetch: Address of next instruction copied from PC to the MAR; instruction copied to MDR; PC incremented to hold address of next instruction.
  • Decode: MDR is copied into the CIR; opcode and operand are decoded.
  • Execute: The instruction is carried out.
  • Each stage is designed to happen concurrently to maximise resources (clock ticks and memory).

Certain central processing units (CPUs) use parallel processing and caching to improve performance. Explain parallel processing and caching in a CPU. [5]
▶ Mark Scheme
Parallel processing (up to 3 marks):
  • Multiple processors/cores used simultaneously
  • Different parts of a task are processed at the same time
  • Reduces the time taken to complete processing-intensive tasks
  • Requires tasks that can be broken into independent sub-tasks
Caching (up to 2 marks):
  • Small, very fast memory located between CPU and RAM
  • Holds frequently or recently used instructions/data
  • Reduces access time compared to fetching from RAM

Secondary Storage
2017 Q2 2017 10 marks ▼
Compare the functional characteristics of Hard Disc Drives (HDD) and Optical Drives and give a typical use and storage capacity for each. [10]
▶ Mark Scheme
HDD (max 4 marks):
  • Magnetic medium; stores data on platters; read/write arm with head moves as disk spins
  • Speed measured in RPM (e.g. 7200 RPM)
  • Fragmentation can slow access; can be hybrid with SSD
  • Typical capacity: 500 GB – 4 TB; Typical use: storing OS, programs, server backup
Optical Drive (max 4 marks):
  • Uses lasers to read/write; burns microscopic indentations (pits and lands) in spiral pattern
  • DVD uses two layers; Blu-ray uses shorter blue wavelength laser
  • Typical capacity: 650 MB – 128 GB; Typical use: movies, music, distribution
Comparison (up to 2 marks): speed, storage capacity, cost, durability, portability
2019 Q4 2019 10 marks ▼
Files A and B are stored on an external hard disk drive (HDD). (a) Describe the functional characteristics of a HDD. [5] (b)(i) Explain why there would be a difference in disk access speeds when loading File A and File B into main memory. [3] (b)(ii) An alternative secondary storage medium which does not have the same issue is a Solid State Drive (SSD). Explain why this is the case. [2]
▶ Mark Scheme
(a) HDD characteristics (5 marks):
  • Magnetic medium; data stored on rotating platters
  • Read/write head on arm moves across disk surface
  • Access time depends on seek time and rotational latency
  • Speed measured in RPM; data stored in tracks and sectors
(b)(i) 3 marks:
  • File A is contiguous (not fragmented); File B is fragmented across the disk
  • Read/write head must move to multiple locations to read fragmented file
  • More seek time and rotational latency → slower access
(b)(ii) 2 marks:
  • SSD has no moving parts; uses flash memory
  • Access is electronic not mechanical → no seek time; fragmentation does not affect speed
2023 Q4 2023 9 marks ▼
(a) Explain, with reference to the functional characteristics of a Hard Disk Drive (HDD), how performance is affected by data fragmentation. [6] (b) Describe how the effect of fragmentation can be reduced on a HDD. [3]
▶ Mark Scheme
(a) 6 marks:
  • HDD stores data on magnetic platters using read/write heads on a moving arm
  • Files are stored in sectors/tracks; over time files are split across non-contiguous sectors (fragmentation)
  • Read/write head must move to multiple locations → increased seek time and rotational latency
  • More head movements → slower file access/loading times
(b) 3 marks — Defragmentation:
  • Defragmentation utility reorganises files so they are stored in contiguous sectors
  • Reduces head movement → faster access times
  • Should be run periodically; not needed/recommended for SSDs

Describe the principles of data compression algorithms. [6]
▶ Mark Scheme
Award 1 mark each (up to 6):
  • Compression reduces the number of bits needed to store or transmit data
  • Lossless compression: original data can be perfectly reconstructed after decompression
  • Lossy compression: some data is permanently discarded; smaller file size but cannot be restored
  • Run-length encoding (RLE): repeated values replaced with count + value; good for images with large uniform areas
  • Huffman coding: more frequent characters given shorter bit codes; less frequent given longer codes
  • Dictionary-based compression: repeated strings replaced with shorter references to a dictionary
  • Reduces storage requirements and transmission time
