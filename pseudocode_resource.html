Sequence

Statements execute in the exact order they are written — top to bottom, one after another.

9.10.1

Sequence is the most fundamental construct. Every line runs in order. No skipping, no repeating — just step-by-step execution. In pseudo-code, sequence means your DECLARE, INPUT, PROCESS and OUTPUT steps happen in a logical order.

DECLARE totalPrice AS REAL
DECLARE tax AS REAL
SET tax = 0.20
INPUT totalPrice
totalPrice = totalPrice * (1 + tax)
OUTPUT "Price with tax: ", totalPrice

⚠️ Exam tip: If you swap lines, the algorithm breaks. Always check your order makes logical sense.

🔀

Selection

The algorithm makes a decision — it checks a condition and takes different paths based on the result.

9.10.1

Selection uses IF...THEN...ELSE...END IF. The condition is evaluated — if TRUE, one branch runs; if FALSE, the other runs. This is how algorithms make decisions.

IF num MOD 2 = 0 THEN
    OUTPUT num, " is an even number"
ELSE
    OUTPUT num, " is an odd number"
END IF

⚠️ Exam tip: Always close with END IF. Nested selections need each one closed separately.

🔄

Repetition

A block of code runs multiple times. Use FOR (fixed count), WHILE (condition-controlled), or REPEAT…UNTIL.

9.10.1

Three loop types: FOR — you know exactly how many times. WHILE — check condition first, may not run at all. REPEAT…UNTIL — runs at least once, checks condition at end.

// FOR loop — fixed repetitions
FOR i = 1 TO numberstocheck
    INPUT num
NEXT i

// WHILE loop — condition controlled
WHILE number <= 0 DO
    INPUT number
END WHILE

// REPEAT...UNTIL — runs at least once
REPEAT
    INPUT num
UNTIL num > 0

⚠️ Exam tip: Choose the right loop type for the problem. Rogue values → WHILE or REPEAT. Known count → FOR.

🔢

Counts

A variable that tracks how many times something has occurred. Must be initialised to zero.

9.10.2

A count variable is declared and set to 0. Each time the tracked condition is met, the counter increments by 1 using count = count + 1.

DECLARE count AS INTEGER
SET count = 0    // Always initialise!

WHILE number != -1 DO
    INPUT number
    IF number > 0 THEN
        count = count + 1
    END IF
END WHILE
OUTPUT "Positive numbers: ", count

⚠️ Exam tip: Forgetting to initialise count to 0 is a common error — examiners look for this.

🚩

Rogue Values

A sentinel value that signals the end of input — allowing an indefinite loop to terminate.

9.10.2

A rogue value (sentinel) is a special input that signals "stop entering data." It must be a value that could never appear in valid data — e.g. -1 for measurements that must be positive, or "quit" for text input.

DECLARE measurement AS REAL
INPUT measurement

WHILE measurement >= 0 DO
    // process measurement here
    OUTPUT measurement * 2.54, " cm"
    INPUT measurement  // re-input inside loop!
END WHILE
// Loop ends when user enters value < 0

⚠️ Exam tip: The INPUT must appear both before the loop AND inside it — otherwise you get an infinite loop.

📦

Variables & Constants

Variables store changing values; constants store fixed values that never change during execution.

9.2.1

Variables — declared with a data type, their value changes during runtime. Constants — set once, never modified. Always use CONSTANT or SET for clarity.

// Constants — fixed values
DECLARE pi AS REAL
SET pi = 3.14       // pi never changes

// Variables — values that change
DECLARE radius AS REAL   // will be inputted
DECLARE area AS REAL     // will be calculated
INPUT radius
area = pi * radius * radius

⚠️ Exam tip: For 9(a) constant questions — identify it by name (e.g. "factor = 2") and explain it never changes during execution.

Mark-Winning Checklist
8 Rules for Full Marks
Based on your teacher's resource guide — every mark point your examiner looks for.
  1. 1
    Declare variables with data types
    Always DECLARE with INTEGER, REAL, STRING, BOOLEAN, ARRAY before using them.
  2. 2
    Initialise variables
    SET count = 0, SET flag = FALSE — initial values prevent logic errors.
  3. 3
    Use INPUT for every user-provided value
    Missing INPUT means data has no way to enter the algorithm.
  4. 4
    Validate inputs and output error messages
    IF radius <= 0 THEN OUTPUT "Error..." — validation is a separate mark point.
  5. 5
    Use the correct loop type
    FOR = known count. WHILE/REPEAT = condition controlled. Rogue value = WHILE with pre-input.
  6. 6
    Show all intermediate calculations
    Don't collapse steps — each calculation line can earn a mark.
  7. 7
    Always OUTPUT your final result
    Forgetting the OUTPUT line at the end is the most common single-mark loss.
  8. 8
    Use self-documenting identifiers
    totalPrice, not x. radiusInMetres, not r. Exception: i, j for loop counters.

CircleCalc.pseudo — Q7 Solution
DECLARE r AS REAL      // ✓ declared with type
DECLARE A AS REAL      // ✓ declared with type
DECLARE C AS REAL      // ✓ declared with type
DECLARE pi AS REAL
SET pi = 3.14           // ✓ constant — value never changes

OUTPUT "Enter radius in metres:"
INPUT r                  // ✓ INPUT for user data

IF r <= 0 THEN           // ✓ validation (1 mark)
    OUTPUT "Error: radius must be greater than 0"
ELSE
    A = pi * r * r         // ✓ calculation (1 mark)
    A = A * 10000          // ✓ convert m² to cm² (1 mark)
    C = 2 * pi * r         // ✓ circumference (1 mark)
    C = C * 100            // ✓ convert m to cm
    OUTPUT "A = ", A, " cm²"  // ✓ correct output
    OUTPUT "C = ", C, " cm"
END IF                    // ✓ always close IF
📋 MARK POINTS:
  • Variables declared with appropriate data types (REAL)
  • pi set as a constant (3.14) — never reassigned
  • INPUT r — user provides the radius
  • Validation: IF r <= 0 with error message
  • Correct area formula: pi × r × r
  • Conversion to cm²: multiply by 10,000
  • Correct circumference: 2 × pi × r, multiply by 100
  • OUTPUT statements for both A and C
LoginSystem.pseudo — Q7 Solution
DECLARE loginDetails[3,5] AS STRING  // 2D array
DECLARE username AS STRING
DECLARE password AS STRING
DECLARE flag AS BOOLEAN
SET flag = FALSE    // ✓ initialised — important!

OUTPUT "Please enter username"
INPUT username
OUTPUT "Please enter password"
INPUT password

FOR i = 0 TO 4       // ✓ loops through 5 users (0-4)
    IF username = loginDetails[0,i] AND password = loginDetails[1,i] THEN
        OUTPUT "Welcome, ", loginDetails[2,i]  // full name
        SET flag = TRUE
    ELSE
        IF username = loginDetails[0,i] AND password <> loginDetails[1,i] THEN
            OUTPUT "Incorrect password"
        END IF
    END IF
NEXT i

IF flag = FALSE THEN
    OUTPUT "Username not found"
END IF
📋 MARK POINTS:
  • 2D array loginDetails[3,5] declared as STRING
  • Boolean flag variable declared and initialised to FALSE
  • Separate INPUT for username and password
  • FOR loop iterating through all 5 records (0 to 4)
  • Condition checks BOTH username AND password match
  • Welcome message including full name (column 2)
  • Separate error for wrong password vs username not found
  • flag = TRUE on success; checked after loop for "not found"
PrimeChecker.pseudo — Q10 Solution
DECLARE Num AS INTEGER
DECLARE Divisor AS INTEGER
DECLARE isPrime AS BOOLEAN

SET isPrime = TRUE    // ✓ assume prime until proven otherwise
SET Divisor = 2       // ✓ start checking from 2

INPUT Num

IF Num < 3 OR Num > 999 THEN  // ✓ validation: 3 to 999
    OUTPUT "Error: Enter a number between 3 and 999"
ELSE
    REPEAT
        IF Num MOD Divisor = 0 THEN  // ✓ divisible? not prime
            SET isPrime = FALSE
        END IF
        SET Divisor = Divisor + 1    // ✓ try next divisor
    UNTIL (isPrime = FALSE) OR (Divisor = Num)  // stop early

    IF isPrime = TRUE THEN
        OUTPUT Num, " is a prime number"
    ELSE
        OUTPUT Num, " is NOT a prime number"
    END IF
END IF
BinaryDenary.pseudo — Q9 Solution (8 marks)
Start Procedure BinaryToDenary
    DECLARE binaryNo[0..7] AS STRING   // 8-char string
    DECLARE denaryNo AS INTEGER
    DECLARE multiplier AS INTEGER

    SET denaryNo = 0
    SET multiplier = 128  // ✓ start with 2^7

    INPUT binaryNo

    IF LEN(binaryNo) <> 8 THEN   // ✓ validate 8 bits
        OUTPUT "Error: Must be exactly 8 bits"
    ELSE
        FOR i = 0 TO 7              // ✓ process each bit
            denaryNo = denaryNo + (INT(binaryNo[i]) * multiplier)
            multiplier = multiplier / 2   // ✓ halve each time
        NEXT i
        OUTPUT "Denary value: ", denaryNo
    END IF
End Procedure

Before You Write: Planning Scaffold
1️⃣

Identify your variables

What data do you need to store? Assign types: INTEGER / REAL / STRING / BOOLEAN. Include a flag if checking match/success. Include a count if counting occurrences.

2️⃣

Plan your loop type

Fixed number of items → FOR...TO...NEXT. Unknown count with sentinel → WHILE...DO / END WHILE with INPUT before AND inside loop. Run at least once → REPEAT...UNTIL.

3️⃣

Plan your validation

What values are invalid? Write the condition that catches them. Always output a descriptive error message. This is usually worth 1 mark by itself.

Plan your calculations

Write each formula step separately. Don't collapse two steps into one. Each line can earn a mark. Check units match the expected output.

5️⃣

Plan your output

What does the question say must be displayed? String literals in quotes + variable. Don't forget the OUTPUT after a successful calculation or loop.

EuroTravel Currency Converter

8 marks~12 min

EuroTravel is a travel agency offering customers the option to purchase foreign currency. Assume the conversion from pounds (£) to euros (€) is £1.00 = €1.14.

EuroTravel wants to enter a value in pounds and provide its customers with a conversion into euros for each value £5 below and £5 above the initial value input. For example, if the user inputs £500.00, the algorithm outputs conversions from £495.00 to £505.00.

Write an algorithm for EuroTravel using pseudo-code. Your algorithm should output a suitable error message for any data entered that is not a number, and be written using self-documenting identifiers.

[ 8 marks ] — Check your answer covers: declarations, validation, loop, calculation ×11, output, self-documenting names

✓ Mark Scheme — EuroTravel [8 marks]

DECLARE poundsInput AS REAL
DECLARE euroRate AS REAL
DECLARE startValue AS REAL
DECLARE currentPounds AS REAL
DECLARE currentEuros AS REAL
SET euroRate = 1.14

OUTPUT "Enter amount in pounds:"
INPUT poundsInput

IF poundsInput is not a number THEN
    OUTPUT "Error: Please enter a valid number"
ELSE
    SET startValue = poundsInput - 5
    FOR i = 0 TO 10
        currentPounds = startValue + i
        currentEuros = currentPounds * euroRate
        OUTPUT "£", currentPounds, " = €", currentEuros
    NEXT i
END IF
  • Variables declared with appropriate types (REAL) [1]
  • euroRate constant set to 1.14 [1]
  • INPUT poundsInput [1]
  • Validation with error message for non-numeric input [1]
  • Loop runs 11 times (£5 below to £5 above = 11 values) [1]
  • Correct calculation: currentPounds × 1.14 [1]
  • Output showing both £ and € values [1]
  • Self-documenting identifiers used throughout [1]

Inches to Metric Converter

8 marks~12 min

A builders' merchant wants to convert imperial measurements (inches) into metric. Conversions: 1 inch = 2.54 cm and 1 inch = 0.0254 m.

Write an algorithm using pseudo-code that allows a user to input:

  • the imperial measurement (in inches)
  • whether the output is to be in cm or m

As the number of values is unknown, your algorithm should terminate when a rogue value less than 0.0 is entered. Your algorithm should output a suitable error message for non-real input, and use self-documenting identifiers.

[ 8 marks ] — Key marks: rogue value loop with pre-INPUT, unit selection selection, two calculations, validation, output

✓ Mark Scheme — Inches Converter [8 marks]

DECLARE inches AS REAL
DECLARE unitChoice AS STRING
DECLARE result AS REAL

INPUT inches     // first input BEFORE the loop

WHILE inches >= 0.0 DO
    IF inches is not a number THEN
        OUTPUT "Error: Please enter a valid real number"
    ELSE
        OUTPUT "Output in cm or m? Enter cm or m:"
        INPUT unitChoice

        IF unitChoice = "cm" THEN
            result = inches * 2.54
            OUTPUT inches, " inches = ", result, " cm"
        ELSE IF unitChoice = "m" THEN
            result = inches * 0.0254
            OUTPUT inches, " inches = ", result, " m"
        ELSE
            OUTPUT "Error: Enter cm or m"
        END IF
    END IF
    INPUT inches     // re-input at END of loop body
END WHILE
  • Variables declared with REAL / STRING types [1]
  • INPUT inches BEFORE the WHILE loop [1]
  • WHILE loop with condition inches >= 0.0 [1]
  • INPUT inches again at the END of the loop body [1]
  • Selection for cm vs m choice [1]
  • Correct cm calculation: × 2.54 [1]
  • Correct m calculation: × 0.0254 [1]
  • Suitable error message for invalid data [1]

Explain the Role of MOD

3 marks~4 min

The following algorithm checks if numbers entered are even or odd:

FOR i = 1 To numberstocheck
    INPUT num
    IF num MOD 2 = 0 Then
        OUTPUT num & " is an even number"
    ELSE
        OUTPUT num & " is an odd number"
    END IF
NEXT i

Explain the role of MOD in the algorithm above. [3 marks]

✓ Mark Scheme — Explain MOD [3 marks]

  • MOD returns the remainder after dividing one number by another [1]
  • In this algorithm, it divides num by 2 to find the remainder [1]
  • If the remainder equals 0, the number is even; otherwise (remainder = 1) it is odd [1]

Example: 7 MOD 2 = 1 (odd), 8 MOD 2 = 0 (even)

Standard Modules

4 marks~5 min

Giving an example, describe standard modules and their benefits. [4 marks]

✓ Mark Scheme — Standard Modules [4 marks]

  • A standard module (also called a procedure/function/subroutine) is a pre-written, named block of code that performs a specific task [1]
  • It can be called/used multiple times from different parts of a program without rewriting the code [1]
  • Example: a module to validate a password, calculate a total, or display a menu — called whenever that task is needed [1]
  • Benefits: reduces code repetition / easier to debug / easier to maintain / improves readability / code can be shared between programs [1 — any valid benefit]
Examiner's Perspective
Common Mistakes to Avoid
❌

Forgetting END IF / END WHILE

Every IF block must end with END IF. Every WHILE must close with END WHILE. Missing these costs marks directly.

❌

INPUT only before the loop

For rogue value loops, INPUT must appear BOTH before the WHILE and at the END of the loop body — otherwise infinite loop.

❌

No validation or error message

Questions that say "output a suitable error message" award a specific mark for that. Don't skip it.

⚠️

Single-letter variable names

Only i and j are acceptable as loop counters. Everything else needs a descriptive name: totalPrice, radiusInMetres, etc.

⚠️

Not initialising count/flag

SET count = 0 and SET flag = FALSE must appear BEFORE the loop. Examiners check initialisation explicitly.

✅

Show every calculation step

Split multi-step calculations: A = pi * r * r, then A = A * 10000. Each intermediate step can earn its own mark.
