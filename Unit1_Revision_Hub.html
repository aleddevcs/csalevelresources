Network Communication Hardware

NIC (Network Interface Card)

Allows a device to connect to a network by converting data into a suitable format. Features a unique MAC address essential for device identification. Can be wired or wireless.

Router

Directs data packets between networks. Acts as a gateway to the Internet. Uses Routing Tables for pathfinding and NAT to manage public/private IP addresses.

Switch

Operates at Layer 2 (Data Link). Connects devices on a LAN and forwards data based on MAC addresses. Supports full-duplex, reducing collisions by sending/receiving simultaneously.

Hub (Legacy)

Operates at Layer 1 (Physical). Broadcasts data to all connected devices regardless of destination, making it inefficient and prone to data collisions.

Wireless Access Point (WAP)

Extends a wired network by allowing wireless devices to connect via Wi-Fi standards (802.11). It acts as a bridge between wireless and wired mediums.

Gateway

A translator between different communication protocols. Often operates at the Application Layer to link incompatible network types (e.g., industrial to corporate).

Bridge

Connects network segments to form a single LAN. It uses MAC addresses to isolate traffic within each segment, reducing overall network congestion.

02

Protocols & Standards

Essential Network Protocols

TCP/IP: The foundational suite. TCP ensures reliable delivery (connection setup, packets); IP handles routing and addressing.

UDP: Simpler, faster alternative for time-sensitive apps (VoIP, Video Streaming) without error checking.

HTTP / HTTPS: Web page transmission. HTTPS encrypts data via TLS/SSL.

SMTP / IMAP / POP3: Email protocols. SMTP sends; IMAP/POP3 retrieve (IMAP syncs across multiple devices).

DHCP: Automatically assigns IP addresses to devices joining a network.

SIP: Initiates and manages multimedia real-time sessions like VoIP calls.

Standard Organizations

  • IEEE

  • IETF

    Internet Engineering Task Force

    Develops Internet standards: TCP/IP, HTTP, SIP.

  • ISO

    International Organization for Standardization

    Broad standards including the global OSI model.

Cellular Networks (Mobile)

Generation Comparison

2G (GSM): Voice and basic SMS only.
2.5G (GPRS): Enhancement for packet-switched data (basic web).
3G: Faster data for web and video calling.
4G (LTE): High-speed streaming and mobile broadband.
5G: Ultra-fast, low latency, massive IoT support.

The SMS Transmission Journey

  • 1. Encode: Handset encodes text into Binary.
  • 2. Transmit: Handset antenna sends radio signal to nearby cell tower.
  • 3. Relay: Tower sends signal via underground/ocean Fiber Cables to a tower near the recipient.
  • 4. Receive: Recipient tower broadcasts radio waves to handset.
  • 5. Decode: Recipient handset antenna converts waves back to binary and decodes as text.

Transmission Media

Inexpensive. Max 100 Mbps. Limited to 100m distance.

Up to 1 Gbps (100m) or 10 Gbps (55m). Reduced crosstalk.

CAT6a

Augmented. Reliable 10 Gbps over full 100m distance. Bulky/Costly.

Fibre Optics (FTTC vs FTTP)

FTTC (Cabinet)

Fibre to the street; copper "last mile". Slower than full fibre.

FTTP (Premises)

Direct fibre to building. Ultra-fast, immune to EMI, long distance.

09

The Hybrid Network Approach

Advantages

  • • Flexibility: Optimal performance based on needs (Fibre speed vs Wireless mobility).
  • • Cost-Effectiveness: Use wireless where cabling installation is too expensive.
  • • Redundancy: Provides failover options if one medium fails (e.g., cellular backup).

Disadvantages

  • • Complexity: Requires sophisticated hardware and configuration knowledge.
  • • Variable Performance: Different media lead to inconsistent speeds across the network.
  • • Security Risks: Wireless signals can be intercepted more easily than physical cables.

The Need for IPv6

The Limit: IPv4

Uses a 32-bit address space, allowing approx 4.3 Billion addresses. Rapid IoT growth has exhausted this capacity.

The Solution: IPv6

Uses a 128-bit address space, allowing for 2^128 addresses. Trillions of times larger; essential for IoT and 5G scalability.

Defining Machine Learning

Machine Learning (ML) is a subset of Artificial Intelligence (AI) focused on creating algorithms that enable systems to learn from data and make decisions with minimal human intervention.

Relationship to AI

AI is the broad science of simulating intelligence. ML is the specific area that trains systems to recognize patterns rather than following explicit, step-by-step programming.

How it Works

ML algorithms process large datasets to automatically build analytical models. They adapt based on new data without being explicitly reprogrammed.

Benefits

• Automation of complex tasks
• Speed on massive datasets
• Continuous improvement

Challenges

• Dependency on high-quality data
• Hard to interpret "Black Box" models

Ethics

• Risk of reinforcing biases

05

Types of Learning

1. Supervised Learning

Guidance: Full Dataset Labelled.

Model is trained on Correct Answers. It learns to associate patterns with specific labels. Once trained, it identifies new data based on these learned associations.

Fruit Analogy: Images clearly labelled as "Apple" are used for training. Model can then identify an apple in a new image.

2. Unsupervised Learning

Guidance: No Labels Provided.

Model finds patterns or groupings on its own. It uses Clustering to group items with similar features together without predefined labels.

Fruit Analogy: Large collection of fruits given without labels. Model groups apples together based on round shape and colour.

3. Semi-Supervised Learning

Guidance: Small amount labelled, most unlabelled.

Labelled data provides initial guidance. As it predicts unlabelled data, human feedback corrects errors, refining accuracy over time.

Fruit Analogy: Model guesses "Orange". Human corrects: "This is an Apple". Model adjusts and learns from the correction.

Definition of a Robot

A robot is a machine capable of performing a complex series of actions automatically. They are task-performing machines, often programmable by computer, designed for precision and efficiency.

Control Types

  • • Externally Controlled: Operated through an external device sending instructions.
  • • Embedded Control: Autonomously operated via internal control systems.

Key Characteristics

  • • Programmable: Can handle tasks from simple to complex.
  • • Automated: Function with little to no human intervention.
  • • Functional Focus: Primary emphasis is task performance over aesthetics.
02

Main Components

Robotic systems consist of several components working together to sense, process, and act.

CPU (The Brain)

Processes data, executes logic, and coordinates all components.

Sensors

Detect environmental conditions like light, sound, temp, and proximity.

Actuators

Small motors attached to the structure that facilitate physical movement.

End-Effectors

Tools like grippers, welders, or cameras that interact with the world.

Power Supply

Provides energy via batteries, direct connections, or solar panels.

The Program

The logic driving behavior, written in languages like Python or C++.

03

Programming Types

Online Programming

1. Teach Pendant (Drive Through)

Manual operation using a handheld device to guide the robot to exact coordinates.

Best for: High-precision assembly tasks.

2. Teach by Example (Lead Through)

Programmer physically moves the robot; movements are recorded for playback.

Best for: Fluid motions like welding or sealing.

Graphical-Offline Programming (Simulation)

Programs created on computers using simulation software before being uploaded. Minimizes downtime as the robot can work while being programmed.

Advantages:

• Troubleshooting in virtual environments.

• Ideal for complex aerospace or automotive tasks.

\[ P(A|B) = \frac{P(A \cap B)}{P(B)} \]

Where:

• \(P(A|B)\) is the probability of event A given B has occurred.

• \(P(A \cap B)\) is the probability of both events occurring.

• \(P(B)\) is the probability of event B.

Example: Probability of coughing is 5%. If the person has a cold, it jumps to 75%. Used in Spam Filters and Medical Diagnosis.

Waterfall Model

Linear & Sequential

Each phase must be completed before the next begins. Rigid structure: Requirements → Design → Implementation → Testing → Deployment → Maintenance.

Advantages

  • ✔ Simple to manage/understand.
  • ✔ Strong documentation.
  • ✔ Clear deliverables per stage.

Disadvantages

  • ✘ Poor adaptability to change.
  • ✘ Errors found late in testing.
  • ✘ High risk for evolving needs.

Best Use Case:

Government Projects / Payroll Systems

Strict regulatory compliance and stability are prioritized over flexibility. Requirements are unlikely to change once development starts.

02

RAD (Rapid Application Development)

Prototyping & Feedback

User-focused iterative approach. High emphasis on quick prototypes and continuous user involvement over extensive planning.

Advantages

  • ✔ Faster development cycle.
  • ✔ Improves software usability.
  • ✔ Early detection of major issues.

Disadvantages

  • ✘ Neglects non-functional needs.
  • ✘ Poor overall system architecture.
  • ✘ Requires highly skilled users.

Best Use Case:

GUI Apps / E-commerce MVP

Ideal for mobile or web apps where user experience is critical and fast validation of a "Minimum Viable Product" is needed.

03

Agile Development

Collaboration & Sprints

Flexible approach emphasizing teamwork (Scrum). Prioritizes small, frequent functional releases and constant customer involvement.

Advantages

  • ✔ Rapid delivery of solutions.
  • ✔ High adaptability to changes.
  • ✔ Continuous improvement culture.

Disadvantages

  • ✘ Lack of formal documentation.
  • ✘ High pressure on customers.
  • ✘ Scope creep risks.

Best Use Case:

Startups / SaaS / AI Chatbots

Suitable for teams expecting rapid market changes or products needing frequent tuning based on real-world user data.

Minimal Planning

No structured phases; all resources are dedicated to coding immediately. Assumes changes will be managed "on the fly".

Advantages

  • ✔ Fast initial output.
  • ✔ Minimal management overhead.
  • ✔ Ideal for very small projects.

Disadvantages

  • ✘ Extremely high risk of failure.

  • ✘ Often leads to cost overruns.

    Phased Changeover

    The new system is introduced in stages. Each phase is tested and approved before the next section of the old system is replaced.

    Advantages

    • ✔ Gradual, manageable implementation.
    • ✔ Issues isolated to one phase.
    • ✔ Good for very complex projects.

    Disadvantages

    • ✘ Requires system compatibility.
    • ✘ Can take months/years.
    • ✘ Complexity in managing two states.

    Best Use Case:

    Large-scale ERP (SAP) / Online University Portals

    Works well with RAD. Allows iterative improvements without disrupting the entire organisation at once.

    03

    Pilot Changeover

    Deploying the system to one small group (branch or department) first. If successful, it is rolled out company-wide.

    Advantages

    • ✔ Low risk to the whole business.
    • ✔ Feedback from real-world use.
    • ✔ Gradual employee training.

    Disadvantages

    • ✘ Only for multi-branch firms.
    • ✘ Can delay full deployment.
    • ✘ High workload for the pilot group.

    Retail Chains (Supermarkets) / Airline Ticketing

    Test a new POS system in one branch. If there are bugs, only that branch is affected while others continue as normal.

    The old system is stopped and the new one starts immediately. There is no fallback option once the switch occurs.

    Advantages

    • ✔ Fast implementation.

    • ✔ Good if no old system exists.

    Disadvantages

    • ✘ Extremely high risk of failure.
    • ✘ No fallback if bugs occur.
    • ✘ Users may struggle to adapt.

    Best Use Case:

    Startups / Simple Businesses / UI Updates

    01

    What is UCD?

    User-Centred Design (UCD) is an iterative process where designers focus on the users and their needs in each phase of the design process.

    The Purpose

    Products are designed to fit the needs, preferences, and limitations of the user, rather than forcing the user to adapt to the product.

    Why it Matters

    "Changing a product’s design too late can cost ten times more than changing it early." UCD ensures acceptance and reduces training time.

    02

    The Five Principles of UCD

    1. Clear Requirements

    Identify who the users are, what tasks they perform, and where/how the product will be used.

    2. User Feedback

    Avoid making assumptions. Collect regular input from users to improve the product constantly.

    3. Active Involvement

    Users directly test mock-ups and prototypes to spot usability issues as early as possible.

    4. Full Integration

    UCD is part of the entire development cycle, running alongside coding and system design.

    5. Iterative Process

    A repeating loop of Design → Test → Improve until user requirements are fully satisfied.

    03

    The Four Phases of UCD

    1

    Specify Context of Use

    Who are the users? What goals do they have? In what environment?

    2

    Specify Requirements

    Define functional and system needs based on real research, not assumptions.

    3

    Create Design Solutions

    Build mock-ups, wireframes, or prototypes based on the requirements.

    4

    Evaluate Designs

    Test against user needs via expert review and user testing sessions.

    04

    Refining Requirements

    Ethnographic Study

    Observing users in their natural environment without interference to spot real frustrations and workarounds.

    Continual Feedback

    Collecting regular input throughout development to make small, ongoing improvements.

    Contextual Enquiry

    Combining observation with interviews where users explain their actions, preferences, and pain points.

    Usability Testing

    Giving users a working prototype to complete typical tasks while noting problems with layout or flow.

    05

    Tools of the Trade

    Persona

    "Who the user is."

    A fictional character based on real research representing a specific user group (e.g. goals, attitudes, skills).

    Scenario

    "What they experience."

    A story about a 'day in the life' of a persona, highlighting their emotional context and practical challenges.

    Use Case

    "How they use the system."

    A step-by-step technical breakdown of the interactions required for a persona to achieve a specific goal.

    06

    Main Design Considerations

    👁️

    Visibility

    Clear actions and navigation. Users should immediately see where to go next.

    ♿

    Accessibility

    Design for all, including those with disabilities (Alt text, screen readers).

    🔤

    Legibility

    Clear fonts, appropriate sizes, and high contrast for easy reading.

    💬

    Language

    Simple, clear sentences. No jargon. "Log in" vs "Authenticate".

    01

    What is DUX?

    Digital User Experience (DUX) refers to the overall effectiveness and satisfaction a user has when interacting with a digital product across different platforms and devices.

    Design Focus

    It encompasses the look, feel, usability, and performance. Good DUX should feel intuitive, with clear layouts that minimize the learning curve and foster trustworthiness.

    02

    The 5 Core Pillars

    1. Design

    Understands user needs and context. Includes emotional UX—how users and developers feel about the product.

    2. Navigability

    The ease of moving through the interface using logical structures, clear labels, and consistent patterns.

    3. Performance

    The speed and reliability of the product. Directly impacts user engagement and accessibility on slower networks.

    4. Efficiency

    Speed of task completion once familiar. Uses shortcuts, autofill, and minimal redundant clicks to save time.

    5. Compatibility

    Ensures the product works across different devices (mobile, desktop), OS (Windows, iOS, Android), and browsers.

    03

    Modern DUX Scope

    Wearables

    • Privacy/Security: Stronger protection of sensitive health data is required for user trust.
    • Context-Aware: Clean, minimal interfaces due to limited screen space.
    • Interaction Balance: Managing passive (vibrations) and active user control.

    Immersive (VR & AR)

    • User Comfort: Managing motion sickness, sensory overload, and emotional impact.
    • Realism: Spatial sound and realistic visuals reduce the friction between user and system.
    • Navigation: Natural interaction for moving and selecting in 3D environments.
    04

    7 Elements for Evaluation

    1. Aesthetics

    Visual design (color/typography), sound (feedback), and touch (haptics).

    2. Info Architecture

    Organizing content logically via menus, search bars, and labels to reduce cognitive load.

    3. Accessibility

    Equal opportunity for all. Includes alt text, screen readers, and high-contrast modes.

    4. HCI

    Focus on making interactions effective using Anthropomorphic, Cognitive, or Empirical approaches.

    5. Ergonomics

    Designing for physical fit—posture, comfort, strap fit, and monitor placement.

    6. Utility

    The usefulness of features. "Does the system offer the right tools for the user's task?"

    7. Performance

    Responsiveness, load times, uptime, and frame rates (especially in VR/AR).

    The 3 HCI Evaluation Approaches

    Anthropomorphic: Replicating human-like qualities (e.g. chatbots) to create emotional connections.

    Cognitive: Designing based on human perception limits (e.g. simple layouts and memory aids).

    Empirical: Usability testing through observation, A/B testing, and raw data collection.

    05

    DUX Success Metrics

    Task Success Rate

    Percentage of users who complete a task successfully.

    Completion Time

    Average time taken to interact with a system or task.

    Retention Rate

    Percentage of users who return to use the product again.

    Conversion Rate

    Users who complete a goal action (e.g. signup or purchase).

    Error Rate

    Errors divided by total attempts (e.g. incorrect button clicks).

    Satisfaction

    Measured via Likert scales, questionnaires, and interviews.

    06

    Jakob Nielsen's 10 Heuristics

    Expert reviews using established usability principles to detect design flaws early.

    01

    Visibility of system status: Users should always be informed about what is going on (e.g. loading indicators).

    02

    Match between system and real world: Use words, phrases, and concepts familiar to the user, not tech jargon.

    03

    User control and freedom: Provide a clear "emergency exit" to leave an unwanted state (e.g. Undo/Redo).

    04

    Consistency and standards: Follow platform conventions so users aren't confused by different words/actions.

    05

    Error prevention: Eliminate error-prone conditions or present a confirmation option before an action.

    06

    Recognition rather than recall: Minimize memory load by making objects and actions visible (e.g. menus vs typing commands).

    07

    Flexibility and efficiency of use: Accelerators (shortcuts) for experts to speed up frequent actions.

    08

    Aesthetic and minimalist design: Dialogues should not contain irrelevant or rarely needed information.

    09

    Recognize, diagnose, and recover from errors: Error messages should be expressed in plain language and suggest a solution.

    10

    Help and documentation: Provide searchable information to help users complete their tasks.
